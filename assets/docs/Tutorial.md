# 사용 튜토리얼

## 하드웨어 요구 사항

반드시 PC 클라이언트 `1920*1080` 해상도 창 모드 또는 전체 화면으로 게임을 실행해야 합니다. (HDR 미지원)

> **왜 PC 버전을 사용해야 하나요?** 스타레일은 모바일 전용 게임과 달리, PC 버전에서 컴퓨터 하드웨어 성능을 더 잘 활용할 수 있습니다.
>
> 에뮬레이터(앱플레이어) 실행은 끊김 현상(렉), 높은 성능 소모 등의 단점이 있으며, 버전 업데이트 때마다 클라이언트를 두 개씩 업데이트하는 번거로움을 겪고 싶지는 않으실 겁니다.
>
> 만약 백그라운드 실행이 필요하다면 [원격 로컬 멀티 유저 데스크톱](/assets/docs/Background.md)을 사용하여 한 대의 컴퓨터에서 여러 바탕화면을 동시에 활성화할 수 있습니다.


## 다운로드

[Releases](https://github.com/moesnow/March7thAssistant/releases/latest)를 클릭하여 다운로드 페이지를 열고 바닥글까지 스크롤합니다.

이름이 `March7thAssistant_full.zip(7z)`와 유사한 파일을 찾아 다운로드합니다.

> March7thAssistant는 타사 서비스인 [Mirrorchyan](https://mirrorchyan.com/?source=m7a-faq) (오픈 소스 커뮤니티를 위한 유료 콘텐츠 배포 플랫폼)과 연동되어 있습니다.
>
> (참고: 중국 본토 다운로드 관련 내용이므로 한국 사용자는 GitHub Releases를 주로 사용하시면 됩니다.)

## 압축 해제

다운로드한 `March7thAssistant_full.zip(7z)` 파일을 찾아 우클릭 후 "압축 풀기"를 선택합니다.

> 반디집 등의 압축 프로그램을 설치했다면 "여기에 풀기" 또는 "알아서 풀기" 등을 선택할 수 있습니다.

## 실행

압축 해제 후 생성된 `March7thAssistant` 폴더를 찾아 더블 클릭하여 엽니다. 내부에는 최소한 다음 파일(폴더)들이 포함되어 있어야 합니다:

- `March7th Launcher.exe`
- `March7th Assistant.exe`
- `March7th Updater.exe`
- `assets`
- `libraries`

`March7th Launcher.exe`를 더블 클릭하여 그래픽 인터페이스(GUI)를 열고, 《면책 조항》에 동의합니다.

> 이 프로그램은 무료 오픈 소스 프로젝트입니다. 만약 돈을 지불했다면 즉시 환불을 요청하세요!!!
>
> 이 프로젝트는 되팔이 행위로 인해 심각한 위협을 받고 있습니다. 우리를 도와주세요!!!
>
> 되팔이에게 지불한 돈은 오픈 소스 자동화 생태계를 더 어렵게 만듭니다. 환불을 요구하고 판매자를 신고해 주세요!!!

## 작업

### 전체 실행 (Full Run)

`일일` → `개척력 소모` → `필드 토벌` → `시뮬레이션 우주` → `망각의 정원/허구 이야기` → `보상 수령` 순서대로 실행됩니다.

이미 완료된 것으로 판단되는 작업은 반복 실행되지 않습니다. 일일 작업과 필드 토벌의 초기화 시간은 매일 새벽 4시이며,

시뮬레이션 우주와 망각의 정원/허구 이야기 등의 초기화 시간은 매주 월요일 새벽 4시입니다.

### 필드 토벌 (토벌런)

[Fhoe-Rail](https://github.com/linruowuyin/Fhoe-Rail) 프로젝트를 호출하여 독립적으로 실행하며, 오늘 완료했는지 여부를 판단하지 않습니다.

사용 전 모든 경계의 닻(텔레포트)을 개방해야 하며, 맵 상에서 이미지 인식을 방해하는 모든 요소(기관새, 잠금 해제 필요한 문 등)를 미리 해결해야 합니다.

성인 여성 모델(원거리 공격, 이동 없음) 캐릭터가 맵 이동에 가장 적합합니다. 예: 아스타, Mar. 7th 등. 근접 캐릭터는 오류가 발생할 수 있으며, 아케론 등의 캐릭터는 전용 토벌 모드를 선택할 수 있습니다.

> 조명 품질은 '중간' 이상, 60프레임을 권장하며, 나머지 그래픽 설정은 영향이 없습니다.

### 시뮬레이션 우주

[Auto_Simulated_Universe](https://github.com/CHNZYX/Auto_Simulated_Universe) 프로젝트를 호출하여 독립적으로 실행하며, 이번 주 완료 여부를 판단하지 않습니다.

매주 34회 실행 후 정지하며, 도중에 닫아도 진행 상황이 저장됩니다. 몰입 보상 수령을 활성화하면 몰입기만 사용하며 개척력은 소모하지 않습니다.

기본 세계: 예를 들어 현재 시뮬레이션 우주 기본값이 '세계 4'인데 '세계 6'을 자동화하고 싶다면, 먼저 게임 내에서 세계 6에 한 번 진입하여 기본 세계 설정을 변경해 주세요.

### 망각의 정원/허구 이야기 (상시 콘텐츠)

독립적으로 실행하며, 이번 주(또는 회차) 완료 여부를 판단하지 않습니다. 자동으로 별(Star) 개수를 인식하며, 만점(3별)일 경우 건너뜁니다.

## 설정

좌측 하단의 `설정`을 클릭하여 어시스턴트 설정 화면으로 진입합니다.

### 프로그램

#### 로그 레벨
일반적으로 기본값인 `간결`로 유지하면 되며, 오류 발생 시 `상세`로 전환하여 문제를 파악할 수 있습니다.

#### 게임 스크린샷

프로그램이 화면을 올바르게 인식하고 있는지 직접 확인할 수 있으며, 영역 선택 후 OCR 문자 인식을 호출하여 던전 이름 등의 특수 문자를 복사하는 데 사용할 수 있습니다.

#### 작업 완료 후

작업이 완료된 후 수행할 동작입니다. `종료`는 게임 종료를 의미합니다.

`반복(Loop)`은 개척력을 기준으로 7일 24시간 무인 반복 실행을 의미합니다(전체 실행 모드에서만 유효).

`컴퓨터 끄기`, `최대 절전 모드`, `절전 모드`, `로그아웃` 전에는 알림이 표시되며 1분간 대기 후 실행됩니다.

#### 반복 실행 재시작 필요 개척력

`작업 완료 후` 옵션이 `반복`으로 설정된 경우에만 유효합니다.

개척력이 설정된 값만큼 회복되면 다시 실행됩니다. 새벽 4시(일일 초기화)는 개척력보다 우선순위가 높습니다.

### 게임

#### 게임 경로

게임을 실행한 상태에서 어시스턴트를 실행하면 자동으로 설정됩니다.

수동으로 설정할 경우 실행기인 `Star Rail\launcher.exe`를 선택하지 않도록 주의하세요.

`Star Rail\Game\StarRail.exe`가 실제 게임 실행 파일입니다.

### 개척력 (Stamina)

#### 던전 유형

개척력 소모 시 돌게 될 던전(고치 등) 유형이며, 개척력을 자동으로 인식하여 횟수를 계산합니다.

#### 던전 이름

개척력 소모 외에도, 일일 훈련 중 `「XXXX」 1회 완료` 임무가 떴을 때

여기서 설정한 던전을 돌며 임무를 완료합니다. 임무가 있어도 완료하고 싶지 않다면 `없음`으로 설정하세요.

#### 지원 캐릭터 사용 활성화

보통 일일 훈련 중 `지원 캐릭터를 사용하여 전투 1회 승리` 임무가 있을 때만 지원 캐릭터를 사용합니다.

이 옵션을 켜면 던전을 돌 때마다 매번 지원 캐릭터를 사용합니다.

#### 전쟁의 여운 활성화

전쟁의 여운 남은 횟수 자동 판단을 지원합니다. 개척력 소모보다 우선순위가 높으며, 초기화 시간은 매주 월요일 새벽 4시입니다.

### 필드 토벌

[Fhoe-Rail](https://github.com/linruowuyin/Fhoe-Rail) 프로젝트를 호출합니다.

중단된 맵부터 계속 실행하려면 `원본 실행`을 클릭하여 토벌런 자체 인터페이스를 연 뒤 디버그 모드를 선택하면 됩니다.

`토벌런 업데이트` 버튼으로 최신 버전으로 한 번에 업데이트할 수 있습니다.

### 시뮬레이션 우주

[Auto_Simulated_Universe](https://github.com/CHNZYX/Auto_Simulated_Universe) 프로젝트를 호출합니다.

빠른 시작 가이드: [프로젝트 문서](https://github.com/Night-stars-1/Auto_Simulated_Universe_Docs/blob/docs/docs/guide/index.md) 

문제 해결 Q&A: [Q&A](https://github.com/Night-stars-1/Auto_Simulated_Universe_Docs/blob/docs/docs/guide/qa.md)

어시스턴트는 `명령줄 사용 방식`을 사용하므로 Python 환경 변수가 올바르게 설정되어 있는지 확인하세요.

운명의 길이나 난이도 등을 수정해야 한다면 `원본 실행`을 클릭하여 시뮬레이션 우주 자체 GUI를 열 수 있습니다.

`시뮬레이션 우주 업데이트` 버튼으로 최신 버전으로 한 번에 업데이트할 수 있습니다.

### 망각의 정원

숫자는 비술(Technique) 사용 횟수를 의미합니다. `-1`은 마지막으로 비술과 일반 공격을 사용하는 캐릭터(즉, 진입 캐릭터)를 의미합니다.

캐릭터에 해당하는 영어 이름은 `March7thAssistant\assets\images\character` 경로에서 확인할 수 있습니다.

### 키 설정 (Keybinding)

키 설정은 어시스턴트 내장 기능(예: 망각의 정원/허구 이야기)에서만 유효합니다.

시뮬레이션 우주나 필드 토벌의 키를 수정해야 하는 경우 해당 프로젝트의 튜토리얼을 확인하세요.

시뮬레이션 우주 `Auto_Simulated_Universe`의 키는 `info.yml`에서 수정할 수 있습니다.

필드 토벌 `Fhoe-Rail` 소스 코드 모드의 `utils` 디렉터리에 이름에 `改按键(키 수정)`이 포함된 `py` 파일이 있으며, 이는 `E`와 `F` 키를 서로 바꾼 것입니다.

## 업데이트

새 버전 알림 팝업이 뜨면 다운로드를 클릭하거나, 수동으로 `March7th Updater.exe`를 더블 클릭하세요.

수동 업데이트를 하려면 `March7thAssistant.zip(7z)`와 유사한 이름의 파일을 다운로드한 뒤,

압축을 풀고 모든 파일을 복사하여 구버전 폴더 안에 붙여넣기(덮어쓰기) 하시면 됩니다.
