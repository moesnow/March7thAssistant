# 利用ガイド

> このドキュメントは簡体字中国語版をもとに AI で翻訳しています。最終更新日: 2026-04-24。差異がある場合は簡体字中国語版を優先してください。

## 実行前の準備

| プラットフォーム | ローカルゲーム | クラウドゲーム |
|------|---------|--------|
| Windows | ✅ | ✅ |
| macOS | ❌ | ✅ |
| Linux | ❌ | ✅ |
| Docker | ❌ | ✅ |

ローカルゲームモードは Windows のみ対応です。PC クライアントを使用し、1920×1080 のウィンドウモードでゲームを実行してください。ウィンドウモードが使えない場合は全画面を使用します。

ゲーム内言語は必ず簡体字中国語に設定してください。また、FPS 監視 HUD、MSI Afterburner、GamePP、HDR、NVIDIA ゲームフィルターなど、画面を覆ったり色味を変えたりするソフトは無効にしてください。

クラウドゲームモードは Windows、macOS、Linux に対応しています。macOS / Linux ユーザーはソースから実行するか、[Docker デプロイ](/assets/docs/Docker.md) を利用できます。

> `クラウド・崩壊：スターレイル` を有効にすると、開拓力消化、デイリー訓練、差分宇宙などのタスクは固定ウィンドウを必要としなくなります。
>
> バックグラウンド実行に対応しますが、フィールド探索と模擬宇宙は引き続き前面全画面が必要です。
>
> バックグラウンド実行や複数デスクトップを使いたい場合は、[リモートローカル多ユーザーデスクトップ](/assets/docs/Background.md) も利用できます。

## ダウンロードとインストール

[Releases](https://github.com/moesnow/March7thAssistant/releases/latest) を開き、ページ下部までスクロールしてください。

`March7thAssistant_full.zip` または `March7thAssistant_full.7z` のような名前のフルパッケージをダウンロードします。

> March7thAssistant はサードパーティサービスの [Mirror酱](https://mirrorchyan.com/?source=m7a-tutorial) を利用しています。
>
> 中国本土でダウンロードが遅い場合は、[Mirror酱 高速ダウンロード](https://mirrorchyan.com/zh/download?rid=March7thAssistant&os=&arch=&channel=stable&source=m7a-tutorial) を試してください。

ダウンロード後、右クリックして「すべて展開」を選ぶか、サードパーティの解凍ソフトを使って展開してください。

展開後のフォルダーには通常、少なくとも次のファイル / フォルダーが含まれます。

- `March7th Launcher.exe`
- `March7th Assistant.exe`
- `March7th Updater.exe`
- `assets`
- `libraries`
- `3rdparty`

`March7th Launcher.exe` をダブルクリックして GUI を起動し、免責事項に同意してください。

Windows タスクスケジューラと併用したい場合や、完全実行を直接開始したい場合は `March7th Assistant.exe` も使用できます。

### コマンドライン起動

`March7th Launcher.exe` はコマンドライン引数に対応しており、GUI を起動しながら特定タスクを自動で開始できます。

Windows でターミナルから実行する場合は、管理者権限で PowerShell、Windows Terminal、または CMD を開いてください。

`Sudo for Windows` に対応している環境なら、`sudo` を使って実行することもできます。

Microsoft のドキュメントによれば、`Sudo for Windows` は Windows 11 バージョン 24H2 以降でのみ利用可能で、

事前にシステム設定から有効化しておく必要があります。詳しくは [Sudo for Windows](https://learn.microsoft.com/zh-cn/windows/advanced-settings/sudo/)

よく使う例：

March7th Launcher.exe -h （ヘルプを表示）

March7th Launcher.exe daily （GUI を起動し、デイリー訓練を実行）

March7th Launcher.exe main -e （タスクが正常終了したら自動で閉じる）

代表的なタスク名には、`main`（完全実行）、`daily`（デイリー訓練）、`power`（開拓力消化）、`currencywars`（貨幣戦争）などがあります。

実行したいタスクは覚えているが名前を忘れた場合は、`March7th Launcher.exe -l` を実行して最新一覧を確認してください。

>
> このプロジェクトは転売行為によって深刻な被害を受けています。転売の通報にご協力ください。

## ホーム画面のタスク

### 完全実行

`完全実行` は日常の無人運用に最も適した入口です。

現在の設定に基づき、`有効化されていて` かつ `更新条件を満たした` 内容だけを実行します。現行バージョンでは、通常次の順番で処理されます。

交換コード受取、デイリー訓練、イベント、歴戦余韻、開拓力消化、貨幣戦争ポイント報酬、差分宇宙ポイント報酬、

フィールド探索、模擬宇宙 / 差分宇宙、混沌の記憶、虚構叙事、末日の幻影、最後の報酬受取。

無効なタスクは自動でスキップされ、すでに完了済み、あるいはまだ次周期に更新されていない内容も再実行されません。

日次更新時刻は既定で毎日午前 4 時、週次コンテンツは通常毎週月曜日の午前 4 時に更新されます。

### 日常

ホーム画面の `日常` メニューには `デイリー訓練` と `開拓力消化` があります。

`デイリー訓練` は未完了の訓練目標を先に判定し、その後で自動実行を試みます。体力を消費して周回するだけなら `開拓力消化` を直接使えば十分です。

`開拓力消化` は設定で選んだダンジョン種類とダンジョン名をもとに自動で挑戦回数を計算し、

予備開拓力、燃料、サポートキャラ、育成目標などのオプションとも連動します。

### 貨幣戦争

現行バージョンは `貨幣戦争・ゼロサムゲーム` に対応しています。ホーム画面から `1回実行` と `ループ実行` を選べます。

設定で `貨幣戦争ポイント報酬` を有効にしている場合、完全実行は週次ロジックに従って報酬の完了状況を自動で確認します。

未完了なら貨幣戦争に入り、今週のポイント処理を行います。

設定画面では `標準対局 / オーバークロック対局`、クイック設定、職級難易度、戦略、速度重視モード、

ポイント報酬受取後に次元装飾品の高速抽出を自動実行するかどうかも設定できます。

### 差分宇宙 / 模擬宇宙

現行バージョンは `差分宇宙・楽園漫記` に対応しています。ホーム画面では `1回実行`、`ループ実行`、`途中引継ぎ` を選べます。

設定で `差分宇宙ポイント報酬` を有効にしている場合、完全実行は週次ロジックに従って報酬完了を確認します。

未完了なら差分宇宙に入り、今週のポイント処理を行います。

設定画面では `通常演算 / 周期演算`、難易度、ステーション優先度、低性能互換モード、報酬受取後の装飾品抽出などを設定できます。

> `模擬宇宙 / 差分宇宙` の設定ページは、通常、毎週上限（エリート100体）まで遺物経験値や霊の珠の涙を周回するために使われます。
>
> 模擬宇宙部分は [Auto_Simulated_Universe](https://github.com/CHNZYX/Auto_Simulated_Universe) を利用していますが、現在は保守が停滞気味です。問題がある場合は差分宇宙の利用を推奨します。

### フィールド探索

フィールド探索は [Fhoe-Rail](https://github.com/linruowuyin/Fhoe-Rail) を利用します。ホーム画面のメニューには `クイック起動`、`原版起動`、`フィールド探索を更新`、`設定ファイルをリセット`、`プログラムフォルダーを開く`、`プロジェクトページを開く` があります。

中断したマップから続けたい場合は `原版起動` で Fhoe-Rail 自体の画面を開き、そのプロジェクト側で適切なモードを選択してください。

使用前に必要なアンカーをすべて開放し、転送や経路探索を妨げる任務は整理しておくのがおすすめです。

大人女性の遠距離・非移動モデルは比較的安定しており、たとえばアスターや三月なのかが該当します。黄泉のようなキャラは専用ルート版と組み合わせられます。

> 60 FPS が必要です。その他の画質設定は通常あまり影響しません。
>
> フィールド探索ルートはすべて有志による手動録画です。改善されたルートや新バージョン向けのマップ共有は歓迎されています。

### 逐光捡金

現行バージョンの `逐光捡金` には次の3種類の挑戦があります。

- `混沌の記憶`
- `虚構叙事`
- `末日の幻影`

この3つは個別に有効化でき、それぞれステージ範囲や2編成のチームを設定できます。週次更新後、完全実行はスイッチと設定に応じて順番に処理します。

特定の1種類だけを個別に実行したい場合は、ホーム画面から直接そのタスクを開始することもできます。

### ツールボックス

左側ナビゲーションの `ツールボックス` には、よく使う補助機能が用意されています。

- `ゲーム画面キャプチャ`：プログラムが正しくゲーム画面を取得できているか確認でき、選択範囲の OCR も行えます。
- `自動会話`：ストーリー会話画面に入ったあと自動で開始します。クラウド・崩壊：スターレイルでは使えません。
- `交換コード`：現在設定されている交換コードを実行します。
- `FPS制限解除`：対応環境で 60 / 120 FPS を切り替えます。
- `タッチモード`：クラウドゲームのモバイル UI 方式でゲームを起動します。[サードパーティプロジェクト](https://github.com/winTEuser/Genshin_StarRail_fps_unlocker) に依存します。

画面認識に問題がある場合は、まず `ツールボックス → ゲーム画面キャプチャ` を開いて、取得された画面が正しいか確認してください。

## 設定ガイド

左下の `設定` をクリックして設定画面に入ります。

### プログラム設定

#### ログレベル

通常は既定の `簡潔` のままで十分です。問題の切り分けが必要な場合だけ `詳細` に切り替えてください。

#### ゲームパス / ランチャーパス

ゲームを起動した後にアシスタントを開けば、通常 `ゲームパス` は自動で検出されます。手動設定する場合は、

ランチャーである `miHoYo Launcher\launcher.exe` を誤って選ばないよう注意してください。実際のゲーム本体は `Star Rail Game\StarRail.exe` です。

HoYoverse ランチャー経由で更新や事前ダウンロードを行いたい場合は、`HoYoPlay パス` も追加で設定してください。

#### タスク完了後

現在のバージョンでは `なし`、`終了`、`シャットダウン`、`スリープ`、`休止`、`再起動`、`ログオフ`、`ディスプレイをオフ`、`スクリプト実行`、`ループ` などに対応しています。

`終了` はゲーム終了を意味します。`スクリプト実行` は別途パス入力が必要です。`シャットダウン` や `スリープ` は実行前に通知が出て、取り消しのための待機時間があります。

#### 定期実行

`ループモード` は旧設定との互換性のために残されていますが、新バージョンでは `ログ` 画面の `定期タスク` を使う方が推奨です。

定期タスクでは次のような内容を設定できます。

- `完全実行`、`開拓力消化`、`貨幣戦争` などの内蔵タスクを起動
- 外部プログラムやスクリプトを起動
- 単一タスクにタイムアウトと強制停止を設定
- 完了後に通知を送信
- 完了後の動作を設定
- 完了後に特定プロセスを自動終了

24時間運用を行う場合は、旧式のループモードではなく `ログ画面の定期タスク` を優先してください。

### 開拓力と日常設定

#### ダンジョン種類 / ダンジョン名

この2つの項目で、開拓力消化時にどのダンジョンを回るかが決まります。プログラムは開拓力に応じて自動で回数を計算します。

`次元分裂` などのイベントを検出した場合も、ここで設定したダンジョンを優先して処理します。

#### 予備開拓力を使用 / 燃料を使用

開拓力消化時に追加で予備開拓力や燃料を使うかを決める項目です。いずれも1回あたりの上限があります。

#### サポートキャラ

`サポートキャラを使用` を有効にすると、必要な場面でサポートを試みます。`サポートキャラを強制使用` を有効にすると、毎回サポートキャラを優先します。

#### 育成目標

`育成目標` を有効にすると、目標に応じて軌跡素材や遺物ダンジョンを自動で選びます。目標を識別できない場合は手動設定した既定ダンジョンに戻ります。

#### 歴戦余韻

有効にすると、完全実行が毎週3回の `歴戦余韻` を優先して処理し、どの曜日から開始するかも設定できます。

### 貨幣戦争設定

毎週の貨幣戦争ポイント報酬だけを自動で回収したい場合は、`「貨幣戦争」ポイント報酬を有効化` だけで十分です。

よく使う項目は次の通りです。

- `種類`：標準対局 / オーバークロック対局
- `クイック設定`
- `職級難易度`
- `貨幣戦争の戦略`
- `特定の詞条が出たら再挑戦を許可`
- `速度重視モードを有効化`

### 差分宇宙 / 模擬宇宙設定

毎週の差分宇宙ポイント報酬だけを自動回収したい場合は、`「差分宇宙」ポイント報酬を有効化` をオンにしてください。

`差分宇宙` ページは主に週次ポイント報酬、図鑑開放、レベル上げを担当し、

通常演算 / 周期演算、難易度、低性能互換モード、報酬後の装飾品抽出、ステーション優先度などを設定できます。

`模擬宇宙 / 差分宇宙` ページは、毎週上限の 100 エリートまで遺物経験値と霊の珠の涙を集めるためのページで、主な項目は次の通りです。

- `実行モード`：統合 / ソースコード（模擬宇宙のみ）
- `種類`：差分宇宙 / 模擬宇宙
- `実行頻度`：毎週 / 毎日
- `実行回数`
- `模擬宇宙の没入報酬を自動受取`
- `運命`（模擬宇宙のみ）
- `難易度`（模擬宇宙のみ）
- `模擬宇宙 GPU 加速を有効化`（模擬宇宙のみ）

`ソースコード` モードを選ぶ場合は、実行環境に利用可能な Python が必要です。

### フィールド探索設定

フィールド探索ページでは、有効化スイッチ以外に次の内容も設定できます。

- `実行モード`：統合 / ソースコード
- `チーム自動切替`
- `マップバージョン`
- `優先惑星`
- `秘技用スナック購入と合成`
- `トークンと期限切れ郵包の購入`

`ソースコード` モードを使う場合は、実行環境に利用可能な Python が必要です。

### 逐光捡金設定

`混沌の記憶`、`虚構叙事`、`末日の幻影` は、それぞれ次の項目を個別に設定できます。

- 有効 / 無効
- ステージ範囲
- 2編成のチーム
- 前回実行時刻

先にチームを整えた上で、週次コンテンツは完全実行に任せるのがおすすめです。

### クラウド・崩壊：スターレイル

`クラウド・崩壊：スターレイルを使用` を有効にすると、アシスタントはブラウザー経由でクラウドゲームを操作し、開拓力消化などのタスクを実行します。

よく使う設定は次の通りです。

- `全画面実行`
- `最大待機時間`
- `ブラウザー種類`
- `国内ミラーでブラウザーとドライバーをダウンロード`
- `ブラウザーデータを保存`
- `ブラウザー画面倍率（DPI）`
- `ウィンドウなしモードを有効化（バックグラウンド実行）`
- `未ログイン時に自動でウィンドウありモードへ切替`

ウィンドウなしモードは模擬宇宙（差分宇宙を除く）とフィールド探索には対応していません。日常や開拓力消化をバックグラウンドで回したい場合は、この一連の設定が重要です。

### その他設定

`その他` ページには、安定性や自動化の体験を高めるための補助スイッチが集まっています。代表的なものは次の通りです。

- `自動戦闘と2倍速検出を有効化`：ゲーム起動前に自動戦闘と2倍速を有効化し、開拓力消化や貨幣戦争などの戦闘中も状態を維持します。
- `自動解像度変更と Auto HDR 無効化`：アシスタント経由でゲームを起動したとき、自動的に 1920×1080 に変更し Auto HDR を無効化します。ローカルモード向けです。
- `ゲームパス自動設定を有効化`：ショートカット、公式ランチャー、実行中プロセスなどからパスを自動判定します。
- `バックグラウンドスクリーンショットを優先`：既定で有効推奨です。オーバーレイや重ね表示による干渉を減らせます。
- `ユーザーログイン時に起動`：タスクスケジューラでログイン後に自動起動します。完全な無人運用には OS 側の自動ログイン設定も必要になる場合があります。

`OCR 加速モード` については、まず `自動` のまま使うのがおすすめです。各モードの概要は次の通りです。

- `自動`（既定）：Windows 10 18362 以降では ONNXRuntime（DirectML）GPU 加速を優先使用します。
- それ以外で OpenVINO が検出されれば OpenVINO CPU を使用し、どちらも使えない場合は ONNXRuntime CPU に戻ります。
- `GPU (ONNXRuntime DirectML)`：DirectML GPU 加速を強制使用します。環境非対応時は自動で戻ります。
- `CPU (OpenVINO)`：OpenVINO CPU 推論を強制使用します。CPU 環境では ONNXRuntime CPU より速いことがありますが、
- **メモリ使用量が継続的に増える問題** があり得ます。アシスタントは定期的にエンジンを再初期化してメモリを解放し、空きメモリが少なくなると ONNXRuntime CPU に切り替えます。
- `CPU (ONNXRuntime)`：ONNXRuntime CPU 推論を強制使用します。互換性が最も高く、メモリ挙動も最も安定しています。
- 認識異常、クラッシュ、メモリ使用量過多がある場合は、まず `CPU (ONNXRuntime)` に切り替えて確認してください。

### キー設定

キー設定は `その他` ページにあり、アシスタント内蔵機能にのみ適用されます。秘技、マップ、ワープ、タスク停止などのショートカットを設定できます。

その中でも `秘技` キーは、開拓力消化や逐光捡金などの内蔵タスクで主に使われます。

模擬宇宙やフィールド探索側のキーを変更したい場合は、各サードパーティプロジェクトのドキュメントや設定ファイルを参照してください。

### アカウント管理（Windows のみ）

`アカウント` ページは Windows でのみ表示され、主にローカルゲームの複数アカウントのエクスポートと切替に使用されます。`クラウド・崩壊：スターレイル` には対応していません。

推奨手順は次の通りです。

1. まずゲームに完全に入り、キャラクターがフィールドに立っている状態まで進めます。
2. `現在のゲームアカウントをエクスポート` をクリックして、現在のアカウントをローカル一覧に保存します。
3. 以後切り替えたいときは、一覧から対象アカウントを選び、`選択したアカウントをインポート` をクリックします。

アカウントページでは、次の操作も可能です。

- `アカウント名変更`：保存済みアカウントに識別しやすい名前を付ける
- `アカウント削除`：保存済みのローカルアカウントデータを削除
- `更新`：ローカルアカウント一覧を再読み込み
- `レジストリをクリア`：現在の端末に残っている関連アカウント情報を削除し、切替不具合の切り分けに使う

`自動ログイン` は、そのアカウントで入力したユーザー名とパスワードも保存し、ログイン期限切れ時に自動入力を試みます。

この機能には一定のリスクがあるため、仕組みを理解している場合にのみ使用してください。

> 重要：ゲームに完全に入る前にアカウントをエクスポートすると、直前のアカウントの UID を誤って読み取ってしまい、保存ファイルと実際のアカウントが一致しないことがあります。
