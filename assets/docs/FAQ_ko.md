# 자주 묻는 질문

> 이 문서는 중국어 간체 원문을 기준으로 AI가 번역했습니다. 마지막 업데이트: 2026-04-24. 차이가 있으면 중국어 간체 원문을 우선해 주세요.

[최신 소식과 튜토리얼은 Bilibili 계정을 참고해 주세요](https://space.bilibili.com/3706960664857075)

### Q: 업데이트가 자주 실패하거나 다운로드가 너무 느립니다. 어떻게 해야 하나요?

A: March7thAssistant는 오픈소스 프로젝트용 유료 콘텐츠 배포 플랫폼인 [Mirror짱](https://mirrorchyan.com/?source=m7a-faq)과 연동되어 있습니다.

업데이트 확인 기능은 무료로 제공되지만, 실제 다운로드 서비스는 유료입니다.

Mirror짱 다운로드 서비스를 구매하지 않더라도, 업데이트가 감지된 뒤 설정에서 해외 소스(GitHub)로 바꾸면 무료로 다운로드할 수 있습니다.

CDK를 구매해 입력하면 네트워크 상태에 덜 영향을 받고 더 빠르고 안정적으로 업데이트할 수 있습니다.

같은 CDK는 MAA처럼 Mirror짱을 사용하는 다른 프로젝트에도 사용할 수 있습니다.

[Mirror짱 고속 다운로드 바로가기](https://mirrorchyan.com/zh/download?rid=March7thAssistant&os=&arch=&channel=stable&source=m7a-faq)

### Q: 실행이 느리거나 실행 파일을 찾을 수 없고, 2147942402 오류가 뜨거나, 백신이 자꾸 파일을 지웁니다. 어떻게 해야 하나요?

A: `어시스턴트 폴더`를 백신 프로그램의 제외 목록 / 화이트리스트 / 신뢰 구역에 추가한 뒤, 패키지를 다시 압축 해제하여 덮어씌우세요.

Windows Defender: `바이러스 및 위협 방지` → `설정 관리` → `제외 사항 추가 또는 제거`. [#373](https://github.com/moesnow/March7thAssistant/issues/373)의 그림도 참고할 수 있습니다.

(`앱 및 브라우저 제어` → `스마트 앱 제어 설정` → `끔`도 필요할 수 있습니다.)

Huorong 보안: `메인 화면` → `오른쪽 위 메뉴` → `신뢰 구역`. 이 [튜토리얼](https://cs.xunyou.com/html/282/15252.shtml)도 참고할 수 있습니다.

### Q: 필터, FPS 표시, 성능 모니터링 프로그램을 켜면 왜 인식이 계속 실패하거나 “화면 전환 실패”가 뜨나요?

A: 이런 프로그램은 게임 화면 위에 추가 텍스트나 시각 효과를 덧씌워서, 어시스턴트의 이미지 인식과 OCR 판정을 망가뜨릴 수 있습니다.

대표적으로 `NVIDIA 게임 필터`, `HDR`, `FPS 모니터링 HUD`, `MSI Afterburner`, `RTSS`, `GamePP` 같은 프로그램,

그리고 `FPS / CPU / GPU`를 띄우는 각종 오버레이가 문제를 일으킬 수 있습니다.

또한 일부 노트북이나 메인보드 제조사 유틸리티에도 비슷한 기능이 있습니다. 예를 들면 `눈 보호 모드`, `비비드 모드`, `게임 강화`, `색상 최적화`, `장면 모드` 등입니다.

이런 기능은 눈에 띄는 오버레이가 없어도 캡처된 화면의 색상, 밝기, 대비를 직접 바꿔 버릴 수 있습니다.

먼저 이런 필터, 하드웨어 모니터링, 성능 측정 프로그램을 완전히 끈 뒤 다시 시도해 보세요.

본인이 익숙해져서 오히려 가장 못 알아차리는 필터가 원인인 경우도 많습니다. 다른 사용자에게서 같은 오류 보고가 대량으로 보이지 않는다면, 우선 자신의 PC 환경부터 점검하는 것이 맞습니다.

로그에 `의심스러운 모니터링 오버레이 텍스트 감지`와 비슷한 문구가 뜬다면 거의 이 문제입니다.

`도구 상자 → 게임 스크린샷`을 열어 어시스턴트가 실제로 받은 화면이 깨끗한지, 오버레이에 가려지지 않았는지도 확인해 보세요.

### Q: 차분 우주 / 시뮬레이션 우주 / 장신구 추출에서 왜 팀을 선택하지 못하거나 특정 캐릭터를 자동 선택하나요? 팀은 어떻게 바꾸나요?

A: 이런 기능은 게임 내부에 저장된 자동 기억 파티를 사용합니다. 필요한 캐릭터가 부족하다고 판단되면 자동으로 `1번 파티`를 시도합니다.

팀을 바꾸고 싶다면 먼저 게임 안에서 원하는 파티를 수동으로 맞춘 뒤, 해당 화면에 한 번 진입했다가 나와서 게임이 기억한 파티가 갱신되었는지 확인하세요.

### Q: 자동 전투가 켜지지 않거나, 포탈과 상호작용하지 않거나, 캐릭터가 엉뚱하게 뛰어다니는 이유는 무엇인가요?

A: 대부분은 `게임 키 설정과 어시스턴트가 가정하는 기본 키가 다르기 때문`입니다. 다음 순서로 확인하세요.

1. 먼저 게임 기본 키를 바꿨는지 확인하세요. 특히 자동 전투(`V`), 상호작용(`F`), 이동(`WASD`)이 중요합니다. 바꿨다면 기본값으로 되돌린 뒤 다시 테스트해 보세요.

2. 그다음 어시스턴트의 `설정 → 기타 → 자동 전투 및 2배속 감지 활성화`가 켜져 있는지 확인하세요.

3. 자동 전투만 문제라면 게임 내부의 `설정 → 전투 기능 → 자동 전투 설정 유지`가 `예`인지도 확인하세요. 필요하면 자동 전투를 한 번 수동으로 켜고 다시 시도해 보세요.

### Q: 작업을 시작한 뒤에는 키보드나 마우스를 쓰거나, 게임을 백그라운드로 보내면 안 되나요?

A: 로컬 게임 모드에서는 대부분의 자동화가 여전히 현재 데스크톱 화면과 입력 상태에 의존합니다. 실행 중에는 키보드와 마우스를 빼앗아 쓰지 않는 것이 좋고, 게임을 백그라운드로 보내는 것도 권장하지 않습니다.

백그라운드 실행이 필요하다면 `클라우드 스타레일`을 우선 고려하세요. 개척력 소모, 일일 훈련, 화폐 전쟁, 차분 우주 같은 작업은 현재 백그라운드 실행을 지원합니다.

필드 파밍과 시뮬레이션 우주는 여전히 전경 전체 화면이 필요합니다.

로컬 모드에서 백그라운드 운용이 필요하다면 [원격 로컬 멀티 사용자 데스크톱](https://m7a.top/#/assets/docs/Background)도 시도해 볼 수 있습니다.

### Q: 전체 실행은 정확히 무엇을 하나요?

A: `전체 실행`은 현재 켜져 있고 새 주기 조건을 만족한 콘텐츠를 자동으로 처리합니다. 보통 다음이 포함됩니다.

리딤 코드, 일일 훈련, 활동, 전쟁의 여운, 개척력 소모, 화폐 전쟁 점수 보상, 차분 우주 점수 보상,

필드 파밍, 시뮬레이션 우주 / 차분 우주, 혼돈의 기억, 허구 이야기, 종말의 환영, 마지막 보상 수령.

이미 끝난 작업이나 아직 초기화되지 않은 작업은 자동으로 건너뜁니다. 기본 일일 초기화 시간은 새벽 4시이고, 주간 콘텐츠는 보통 매주 월요일 새벽 4시에 초기화됩니다.

### Q: 안내 창을 찾을 수 없다고 나오면 어떻게 해야 하나요?

A: 게임 내 언어가 중국어 간체인지 확인하세요.

### Q: 새로 추가된 던전이 없어요. 추광줍기에서 신규 캐릭터가 없어요.

A: 먼저 최신 버전인지 확인하세요. 최신인데도 던전이 없다면 `assets\config\instance_names.json`을 직접 수정해 수동으로 추가할 수 있습니다.

`혼돈의 기억 / 허구 이야기 / 종말의 환영 / 지원`에서 신규 캐릭터가 빠져 있다면,

해당 화면에서 도구 상자의 `스크린샷 캡처`를 사용해 캐릭터 초상화를 선택하고 `선택한 스크린샷 저장`을 누르세요.

그 이미지를 `assets\images\share\character`에 넣고 `assets\config\character_names.json`을 수정하면 됩니다.

압축하지 않은 원본 형태로 [Issue](https://github.com/moesnow/March7thAssistant/issues)에 올려 주셔도 좋고, [PR](https://github.com/moesnow/March7thAssistant/pulls)도 환영합니다.

### Q: 실행 중 메모리 사용량이 계속 증가하거나, 게임이 갑자기 멈추거나 튕기거나, 시스템이 재부팅됩니다. 어떻게 해야 하나요?

A: 메모리 사용량이 비정상적으로 높아지거나 프로그램이 이유 없이 종료되거나, 실행 도중 게임 화면이 불안정해지면 먼저 OCR 가속 모드 호환성을 확인하세요.

기본적으로 어시스턴트는 Windows 10 빌드 18362 이상에서 ONNXRuntime DirectML GPU 가속을 사용합니다.

Linux나 일부 ARM 장치처럼 DirectML을 지원하지 않는 환경에서는 OpenVINO가 감지될 경우 OpenVINO CPU 추론을 자동 사용합니다.

**OpenVINO는 일부 PC에서 메모리 사용량이 지속적으로 증가하는 문제**가 있을 수 있습니다. 어시스턴트는 추론 엔진을 주기적으로 다시 만들면서 메모리를 정리하고,

사용 가능한 물리 메모리가 4GB 미만으로 떨어지면 자동으로 ONNXRuntime CPU로 되돌립니다. 하지만 장치 자체 메모리가 적다면 여전히 눈에 띄는 변동이 생길 수 있습니다.

**우선 OCR 가속 모드를 `CPU (ONNXRuntime)`로 수동 변경해 보는 것을 권장합니다** (`설정 → 기타 → OCR 가속 모드`).

바꾼 뒤 다시 실행해서 메모리 상태와 안정성이 나아지는지 확인하세요. 문제가 사라지면 이전 모드가 현재 환경과 맞지 않았던 것이므로 ONNXRuntime CPU를 계속 사용하면 됩니다.

DirectML GPU 가속에서 크래시나 인식 문제가 생길 때도 먼저 `CPU (ONNXRuntime)`으로 되돌려 확인하는 것이 좋습니다.

### Q: 더블 클릭 후 창이 잠깐 떴다가 사라지고, GUI가 열리지 않거나, 터미널에서 실행해도 창이 바로 닫힙니다. 어떻게 해야 하나요?

A: 이런 현상은 보통 한 번에 묶어서 점검할 수 있습니다.

`March7th Launcher.exe`나 다른 실행 파일을 더블 클릭했는데 터미널 창이 번쩍 뜨고 사라지거나 GUI가 전혀 열리지 않는다면,

터미널에서 직접 실행해 실제 오류 메시지를 보는 것이 좋습니다.

관리자 권한으로 PowerShell, Windows Terminal, 또는 CMD를 연 뒤, 어시스턴트 폴더로 이동해서 해당 명령을 실행하세요.

예를 들어:

PowerShell:

```powershell
& '.\March7th Launcher.exe' -S
```

CMD:

```cmd
"March7th Launcher.exe" -S
```

시스템이 [Sudo for Windows](https://learn.microsoft.com/zh-cn/windows/advanced-settings/sudo/)를 지원한다면 `sudo`를 써도 됩니다.

Microsoft 문서 기준으로 `Sudo for Windows`는 Windows 11 24H2 이상에서만 사용할 수 있으며, 사용 전에 시스템 설정에서 미리 활성화해야 합니다.

이 방식으로 실행했을 때 오류가 보인다면, 우선 표시된 실제 오류 메시지를 기준으로 점검하세요.

대표적인 원인은 권한 부족, 백신에 의한 파일 삭제, 의존성 차단, 잘못된 작업 디렉터리입니다.
