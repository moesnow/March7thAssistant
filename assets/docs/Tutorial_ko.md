# 사용 튜토리얼

## 하드웨어 요구사항

| 플랫폼 | 로컬 게임 | 클라우드 게임 |
|------|---------|-----------|
| Windows | ✅ | ✅ |
| macOS | ❌ | ✅ |
| Linux | ❌ | ✅ |

로컬 구동 시 반드시 **PC 클라이언트 1920×1080 해상도 창 모드**(창 모드가 불가한 경우 전체 화면)로 게임을 실행해야 합니다.

또한 프레임 모니터링 HUD, MSI Afterburner, Game++, HDR, NVIDIA 게임 필터 등 **게임 화면에 영향을 줄 수 있는 모든 소프트웨어**를 종료해 주세요.

**macOS / Linux 사용자**: 소스 코드를 통해 클라우드 게임 모드를 사용하거나 [Docker 배포](/assets/docs/Docker.md)를 이용할 수 있습니다.

> **왜 PC 버전을 사용해야 하나요?** 붕괴: 스타레일은 모바일 전용 게임과 달리, PC 클라이언트에서 하드웨어 성능을 온전히 활용할 수 있습니다.
>
> 앱플레이어(에뮬레이터)는 끊김 현상이 발생하고 리소스를 과도하게 점유합니다. 무엇보다 버전 업데이트 때마다 클라이언트를 이중으로 업데이트하는 번거로움을 겪고 싶진 않으시겠죠?
>
> 백그라운드 실행이 필요하다면 [원격 로컬 멀티 데스크톱](/assets/docs/Background.md) 기능을 사용하여 한 PC에서 여러 데스크톱을 동시에 실행해 보세요.

> 중국 내 다운로드 속도가 느리다면 [Mirror 짱 고속 다운로드](https://mirrorchyan.com/zh/download?rid=March7thAssistant&os=&arch=&channel=stable&source=m7a-faq)를 이용해 보세요.

## 압축 해제

다운로드한 `March7thAssistant_full.zip(7z)` 파일을 마우스 오른쪽 버튼으로 클릭하고 "모두 압축 풀기"를 선택합니다.

> 반디집 등 별도의 압축 프로그램을 사용 중이라면 "알아서 풀기" 등을 선택하시면 됩니다.

## 실행

압축 해제된 `March7thAssistant` 폴더를 열면 최소한 다음 파일(폴더)들이 있어야 합니다:

- `March7th Launcher.exe`
- `March7th Assistant.exe`
- `March7th Updater.exe`
- `assets`
- `libraries`

`March7th Launcher.exe`를 더블 클릭하여 실행하고, "면책 조항"에 동의해 주세요.

> 이 프로그램은 **무료 오픈소스 프로젝트**입니다. 만약 돈을 지불하고 구매하셨다면 **즉시 환불을 요청하세요!!!**
>
> 현재 불법 재판매 행위로 인해 프로젝트가 심각한 위협을 받고 있습니다. 여러분의 도움이 필요합니다!!!
>
> 중고 장터 등에서 4,000명 이상의 되팔이들이 활동하고 있습니다. 이들에게 지불된 돈은 오픈소스 생태계를 위협합니다. **반드시 환불받고 판매자를 신고해 주세요!!!**

## 임무

### 전체 실행

`일일` → `개척력` → `필드 정리` → `시뮬레이션 우주` → `빛을 쫓아 금을 줍다` → `보상 수령` 순서로 작업을 수행합니다.

이미 완료된 것으로 판단된 임무는 중복 실행되지 않습니다. 일일 임무와 필드 정리의 리셋 시간은 매일 새벽 4시이며,

시뮬레이션 우주와 빛을 쫓아 금을 줍다의 리셋 시간은 매주 월요일 새벽 4시입니다.

### 필드 정리

[Fhoe-Rail](https://github.com/linruowuyin/Fhoe-Rail) 프로젝트를 호출합니다. 독립적으로 실행되며 당일 완료 여부를 별도로 체크하지 않습니다.

사용 전 반드시 모든 앵커(텔레포트 포인트)를 해금해야 하며, 맵 상에서 이미지 인식을 방해하는 요소(기계 새, 잠금 해제가 필요한 문 등)를 모두 클리어한 상태여야 합니다.

**성인 여성 체형**(원거리 공격, 이동 시 위치 고정) 캐릭터가 맵 이동에 가장 적합합니다. (예: 아스타, Mar. 7th). 근접 캐릭터는 오류가 발생할 수 있으며, 아케론 등 비술 공격 캐릭터는 전용 모드를 선택할 수 있습니다.

> 조명 품질은 '중간' 이상, 프레임은 60fps 설정을 권장합니다. 그 외 그래픽 설정은 영향이 없습니다.

### 시뮬레이션 우주

[Auto_Simulated_Universe](https://github.com/CHNZYX/Auto_Simulated_Universe) 프로젝트를 호출합니다. 독립적으로 실행되며 주간 완료 여부를 체크하지 않습니다.

매주 34회 실행 후 정지되며, 중간에 종료하더라도 진행 상황이 저장됩니다. '몰입 보상 수령'을 활성화하면 몰입기만 사용하고 개척력은 소모하지 않습니다.

기본 세계 설정: 예를 들어 현재 기본 세계가 4인데 세계 6을 자동화하고 싶다면, 먼저 게임 내에서 세계 6에 한 번 진입하여 기본값을 변경해 주세요.

### 빛을 쫓아 금을 줍다

독립적으로 실행되며 주간 완료 여부를 체크하지 않습니다. 획득한 별의 개수를 자동 인식하여 만점이면 실행을 건너뜁니다.

## 설정

좌측 하단의 `설정`을 클릭하여 어시스턴트 설정 화면으로 진입합니다.

### 프로그램

#### 로그 레벨
일반적으로 기본값인 `간결`을 권장합니다. 문제가 발생하여 디버깅이 필요한 경우에만 `상세`로 변경하세요.

#### 게임 스크린샷

프로그램이 인식한 화면이 정상적인지 확인할 수 있습니다. 영역을 드래그하여 OCR 텍스트 인식을 수행할 수 있으며, 던전 이름에 포함된 희귀 한자 등을 복사할 때 유용합니다.

#### 임무 완료 후

모든 임무가 끝난 후 수행할 동작입니다. `종료`는 게임 종료를 의미합니다.

`순환`은 개척력에 맞춰 24시간 무인으로 프로그램을 순환 실행하는 모드입니다(전체 실행 시에만 유효).

`시스템 종료`, `최대 절전`, `절전`, `로그오프` 선택 시 실행 전 알림이 표시되며 1분 후 수행됩니다.

#### 순환 실행 재시작 필요 개척력

`임무 완료 후` 옵션이 `순환`일 때만 유효합니다.

개척력이 설정된 값만큼 회복되면 다시 실행됩니다. 단, 새벽 4시 리셋이 개척력보다 우선순위가 높습니다.

### 게임

#### 게임 경로

게임을 켜둔 상태에서 어시스턴트를 실행하면 자동으로 설정됩니다.

수동 설정 시 런처인 `Star Rail\launcher.exe`를 선택하지 않도록 주의하세요.

반드시 게임 실행 파일인 `Star Rail\Game\StarRail.exe`를 선택해야 합니다.

### 개척력

#### 던전 유형

개척력을 소모하여 파밍할 던전 종류입니다. 현재 개척력을 자동 인식하여 횟수를 계산합니다.

#### 던전 이름

단순 파밍 외에도, 일일 훈련에 `"XXXX" 1회 완료` 임무가 있을 경우

여기서 설정한 던전을 클리어하여 임무를 수행합니다. 해당 임무가 있어도 던전을 돌고 싶지 않다면 `없음`으로 설정하세요.

#### 지원 캐릭터 사용 활성화

기본적으로는 일일 훈련에 `지원 캐릭터를 사용하여 전투 승리 1회` 임무가 있을 때만 지원 캐릭터를 빌립니다.

이 옵션을 켜면 모든 던전 도전 시마다 지원 캐릭터를 사용합니다.

#### 전쟁의 여운 활성화

전쟁의 여운 남은 횟수를 자동으로 파악하여 수행합니다. 개척력 소모보다 우선순위가 높으며, 리셋 시간은 매주 월요일 새벽 4시입니다.

### 필드 정리

[Fhoe-Rail](https://github.com/linruowuyin/Fhoe-Rail) 프로젝트를 호출합니다.

중단된 지점부터 다시 시작하려면 `원본 실행`을 클릭하여 Fhoe-Rail 자체 인터페이스를 열고 '디버그 모드'를 체크한 뒤 원하는 맵을 선택하세요.

`필드 정리 업데이트` 버튼으로 최신 버전으로 간편하게 업데이트할 수 있습니다.

### 시뮬레이션 우주

[Auto_Simulated_Universe](https://github.com/CHNZYX/Auto_Simulated_Universe) 프로젝트를 호출합니다.

빠른 시작 가이드: [프로젝트 문서](https://github.com/Night-stars-1/Auto_Simulated_Universe_Docs/blob/docs/docs/guide/index.md)

문제 해결 Q&A: [Q&A](https://github.com/Night-stars-1/Auto_Simulated_Universe_Docs/blob/docs/docs/guide/qa.md)

어시스턴트는 `명령줄(CLI) 방식`으로 작동하므로 Python 환경 변수 설정이 올바른지 확인해야 합니다.

운명의 길이나 난이도 등을 변경하려면 `원본 실행`을 클릭하여 자체 그래픽 인터페이스에서 설정하세요.

`시뮬레이션 우주 업데이트` 버튼으로 최신 버전으로 간편하게 업데이트할 수 있습니다.

### 망각의 정원

숫자는 비술 사용 횟수입니다. `-1`은 마지막으로 비술을 쓰고 평타로 진입하는 캐릭터(선공 캐릭터)를 의미합니다.

캐릭터별 영문 이름은 `March7thAssistant\assets\images\character` 폴더에서 확인할 수 있습니다.

### 단축키

단축키 설정은 어시스턴트 자체 기능(예: 빛을 쫓아 금을 줍다)에만 적용됩니다.

시뮬레이션 우주나 필드 정리의 단축키를 변경하려면 해당 프로젝트의 문서를 참조하세요.

시뮬레이션 우주(`Auto_Simulated_Universe`)의 단축키는 `info.yml` 파일에서 수정 가능합니다.

필드 정리(`Fhoe-Rail`) 소스 모드의 `utils` 폴더에는 `E`키와 `F`키를 서로 바꾸는 스크립트(`...改按键...py`)가 포함되어 있습니다.

## 업데이트

새 버전 알림 창이 뜨면 다운로드를 클릭하거나, 수동으로 `March7th Updater.exe`를 실행하세요.

수동 업데이트 시에는 `March7thAssistant.zip(7z)` 파일을 다운로드한 뒤,

압축을 풀고 모든 파일을 복사하여 구버전 폴더에 덮어씌우면 됩니다.
