# 사용 튜토리얼

> 이 문서는 중국어 간체 원문을 기준으로 AI가 번역했습니다. 마지막 업데이트: 2026-05-03. 차이가 있으면 중국어 간체 원문을 우선해 주세요.

## 실행 전 준비

| 플랫폼 | 로컬 게임 | 클라우드 게임 |
|------|---------|--------|
| Windows | ✅ | ✅ |
| macOS | ❌ | ✅ |
| Linux | ❌ | ✅ |
| Docker | ❌ | ✅ |

로컬 게임 모드는 Windows만 지원하며, 반드시 PC 클라이언트를 1920×1080 해상도의 창 모드로 실행해야 합니다. 창 모드가 불가능하면 전체 화면을 사용하세요.

게임 내 언어는 반드시 중국어 간체로 설정해야 하며, 프레임 표시 HUD, MSI Afterburner, GamePP, HDR, NVIDIA 게임 필터 등 화면을 가리거나 색을 바꾸는 도구는 모두 꺼 주세요.

클라우드 게임 모드는 Windows, macOS, Linux를 지원합니다. macOS / Linux 사용자는 소스 실행으로 사용할 수 있고, [Docker 배포](/assets/docs/Docker.md)도 사용할 수 있습니다.

> `클라우드 스타레일`을 켜면 개척력 소모, 일일 훈련, 차분 우주 같은 작업은 더 이상 고정된 게임 창을 요구하지 않습니다.
>
> 백그라운드 실행이 가능하지만, 필드 파밍과 시뮬레이션 우주는 여전히 전경 전체 화면이 필요합니다.
>
> 백그라운드 실행이나 다중 데스크톱이 필요하다면 [원격 로컬 멀티 사용자 데스크톱](/assets/docs/Background.md)도 고려해 보세요.

## 다운로드 및 설치

[Releases](https://github.com/moesnow/March7thAssistant/releases/latest)를 열고 페이지 아래쪽으로 내려갑니다.

`March7thAssistant_full.zip` 또는 `March7thAssistant_full.7z`와 비슷한 이름의 전체 패키지를 다운로드하세요.

> March7thAssistant는 제3자 서비스인 [Mirror짱](https://mirrorchyan.com/?source=m7a-tutorial)을 사용합니다.
>
> 중국 본토에서 다운로드가 느리다면 [Mirror짱 고속 다운로드](https://mirrorchyan.com/zh/download?rid=March7thAssistant&os=&arch=&channel=stable&source=m7a-tutorial)를 시도해 보세요.

다운로드가 끝나면 압축 파일을 마우스 오른쪽 버튼으로 눌러 “모두 압축 풀기”를 선택하거나, 별도의 압축 프로그램으로 해제하세요.

압축을 푼 폴더에는 보통 최소한 다음 파일과 폴더가 있어야 합니다.

- `March7th Launcher.exe`
- `March7th Assistant.exe`
- `March7th Updater.exe`
- `assets`
- `libraries`
- `3rdparty`

`March7th Launcher.exe`를 더블 클릭해 GUI를 열고 면책 조항에 동의하세요.

Windows 작업 스케줄러와 함께 쓰거나 전체 실행을 바로 수행하고 싶다면 `March7th Assistant.exe`를 사용할 수도 있습니다.

### 명령줄 실행

`March7th Launcher.exe`는 명령줄 인수를 지원하므로 GUI를 열면서 특정 작업을 바로 시작할 수 있습니다.

Windows에서 터미널로 실행할 때는 관리자 권한으로 연 PowerShell, Windows Terminal, CMD를 사용하는 것이 좋습니다.

시스템이 `Sudo for Windows`를 지원한다면 `sudo`를 사용해도 됩니다.

Microsoft 문서에 따르면 `Sudo for Windows`는 Windows 11 24H2 이상에서만 사용할 수 있으며,

사용 전에 시스템 설정에서 먼저 활성화해야 합니다. 자세한 내용: [Sudo for Windows](https://learn.microsoft.com/zh-cn/windows/advanced-settings/sudo/)

자주 쓰는 예시는 다음과 같습니다.

`March7th Launcher.exe -h` (도움말 보기)

`March7th Launcher.exe daily` (GUI를 열고 일일 훈련 실행)

`March7th Launcher.exe main -e` (작업이 정상 종료되면 자동 종료)

대표적인 작업 이름으로는 `main`(전체 실행), `routine`(일상), `daily`(일일 훈련), `power`(개척력 소모), `currencywars`(화폐 전쟁) 등이 있습니다.

실행하고 싶은 작업은 기억나는데 이름이 생각나지 않는다면 `March7th Launcher.exe -l`로 최신 목록을 확인하세요.

> 이 프로그램은 무료 오픈소스 프로젝트입니다. 돈을 주고 구입했다면 즉시 환불을 요청하세요.
>
> 본 프로젝트는 되팔이 문제로 심각한 피해를 받고 있습니다. 되팔이 신고에 협조해 주세요.

## 홈 화면 작업

### 전체 실행

`전체 실행`은 무인 일상 운용에 가장 적합한 진입점입니다.

현재 설정을 바탕으로 `활성화되어 있고` `새 주기 조건을 만족한` 항목만 실행합니다. 현재 버전에서 전체 실행은 보통 다음 내용을 순서대로 처리합니다.

리딤 코드, 일일 훈련, 활동, 전쟁의 여운, 개척력 소모, 화폐 전쟁 점수 보상, 차분 우주 점수 보상,

필드 파밍, 시뮬레이션 우주 / 차분 우주, 혼돈의 기억, 허구 이야기, 종말의 환영, 마지막 보상 수령.

비활성화된 작업은 자동으로 건너뛰고, 이미 완료되었거나 아직 다음 주기로 초기화되지 않은 내용도 반복하지 않습니다.

기본 일일 초기화 시간은 새벽 4시이며, 주간 콘텐츠는 보통 매주 월요일 새벽 4시에 초기화됩니다.

### 일상

홈 화면의 `일상` 메뉴에는 `일상`, `일일 훈련`, `개척력 소모`가 있습니다.

`일상`은 이 묶음 작업의 초기화 여부를 무시하고, 현재 설정에 따라 리딤 코드, 육성 목표 초기화, 이벤트 감지, 전쟁의 여운, 개척력 소모, 일일 훈련, 마지막 보상 수령까지 한 번에 처리합니다. 자동으로 건너뛰는 것은 비활성화된 항목뿐입니다.

`일일 훈련`은 먼저 아직 완료되지 않은 훈련 목표를 판별한 뒤 자동으로 처리합니다. 체력 던전만 돌고 싶다면 `개척력 소모`를 직접 사용하면 됩니다.

`개척력 소모`는 설정에서 선택한 던전 종류와 던전 이름을 기준으로 자동으로 횟수를 계산하며,

예비 개척력, 연료, 지원 캐릭터, 육성 목표 같은 옵션과도 함께 작동합니다.

개척력 설정에서 `개척력 소모 활성화`를 끄면 `전체 실행`과 `일상`은 전쟁의 여운과 개척력 소모를 함께 건너뜁니다. `개척력 소모` 작업을 단독 실행하는 경우에는 이 스위치의 영향을 받지 않습니다.

### 화폐 전쟁

현재 버전은 `화폐 전쟁·제로섬 게임`을 지원합니다. 홈 화면에서 `한 번 실행`과 `반복 실행`을 선택할 수 있습니다.

설정에서 `화폐 전쟁 점수 보상`을 켜 두었다면 전체 실행이 주간 점수 보상 완료 여부를 자동으로 확인합니다.

완료되지 않았다면 화폐 전쟁에 진입해 이번 주 점수를 처리합니다.

설정 페이지에서는 `표준 대결 / 오버클럭 대결`, 빠른 설정, 직급 난이도, 전략, 속도전 모드,

그리고 점수 보상 수령 후 장신구 빠른 추출 실행 여부까지 설정할 수 있습니다.

### 차분 우주 / 시뮬레이션 우주

현재 버전은 `차분 우주·낙원만유`를 지원합니다. 홈 화면에서 `한 번 실행`, `반복 실행`, `중간 인수`를 선택할 수 있습니다.

설정에서 `차분 우주 점수 보상`을 켜 두었다면 전체 실행이 주간 점수 보상을 확인합니다.

아직 끝나지 않았다면 차분 우주에 진입해 이번 주 점수를 처리합니다.

설정 페이지에서는 `일반 연산 / 주기 연산`, 난이도, 스테이션 우선순위, 저사양 호환 모드,

보상 수령 후 장신구 추출 자동 실행 여부 등을 추가로 설정할 수 있습니다.

> `시뮬레이션 우주 / 차분 우주` 설정 페이지는 주당 100 엘리트 한도까지 유물 경험치와 영광의 눈물을 반복 파밍하는 용도로 주로 사용됩니다.
>
> 시뮬레이션 우주 부분은 [Auto_Simulated_Universe](https://github.com/CHNZYX/Auto_Simulated_Universe) 프로젝트를 사용하지만, 현재 유지보수가 사실상 정체된 상태입니다. 문제가 있으면 차분 우주 사용을 권장합니다.

### 필드 파밍

필드 파밍은 [Fhoe-Rail](https://github.com/linruowuyin/Fhoe-Rail) 프로젝트를 사용합니다. 홈 메뉴에서는 `빠른 시작`, `원본 실행`, `필드 파밍 업데이트`, `설정 초기화`, `프로그램 폴더 열기`, `프로젝트 홈페이지 열기`를 제공합니다.

중단된 위치에서 이어서 실행해야 한다면 `원본 실행`으로 Fhoe-Rail 자체 인터페이스를 열고, 그 프로젝트 안에서 적절한 모드를 선택하세요.

사용 전에는 필요한 모든 앵커를 해금하고, 순간이동이나 길찾기를 방해하는 퀘스트는 정리해 두는 것이 좋습니다.

성인 여성 원거리 비이동형 캐릭터가 보통 가장 안정적입니다. 예를 들면 아스타, 3월 7일이 그렇습니다. 아케론 같은 캐릭터는 전용 경로 버전과 함께 사용할 수 있습니다.

> 60 FPS가 필요합니다. 그 외 그래픽 설정은 대체로 큰 영향을 주지 않습니다.
>
> 모든 필드 파밍 루트는 자원봉사자가 수동으로 녹화해 제공합니다. 새로운 버전의 맵이나 더 나은 루트를 공유해 주시면 큰 도움이 됩니다.

### 추광줍기

현재 버전의 `추광줍기`는 다음 세 가지 챌린지로 구성됩니다.

- `혼돈의 기억`
- `허구 이야기`
- `종말의 환영`

세 항목 모두 각각 독립적으로 활성화할 수 있고, 층 범위와 2파티 편성도 따로 설정할 수 있습니다. 주간 초기화 이후 전체 실행은 사용자의 스위치와 설정에 따라 순서대로 처리합니다.

특정 항목만 따로 돌리고 싶다면 홈 화면에서 해당 작업을 직접 시작해도 됩니다.

### 도구 상자

왼쪽 내비게이션의 `도구 상자`는 다음과 같은 보조 기능을 제공합니다.

- `게임 스크린샷`: 프로그램이 올바른 화면을 가져오고 있는지 확인하고, 선택 영역 OCR도 수행할 수 있습니다.
- `자동 대화`: 스토리 대화 장면에 들어간 뒤 자동으로 시작합니다. 클라우드 스타레일에서는 지원하지 않습니다.
- `리딤 코드`: 현재 설정된 교환 코드를 실행합니다.
- `FPS 잠금 해제`: 지원되는 플랫폼에서 60 / 120 FPS를 전환합니다.
- `터치 모드`: 클라우드 게임 모바일 UI 방식으로 게임을 시작하며, [서드파티 프로젝트](https://github.com/winTEuser/Genshin_StarRail_fps_unlocker)에 의존합니다.

화면 인식 문제가 있다면 가장 먼저 `도구 상자 → 게임 스크린샷`을 열어 실제 캡처된 화면이 올바른지 확인하세요.

## 설정 설명

왼쪽 아래의 `설정`을 눌러 설정 화면으로 들어갑니다.

### 프로그램 설정

#### 로그 레벨

보통은 기본값인 `간결`이면 충분합니다. 문제를 추적해야 할 때만 `상세`로 바꾸세요.

#### 게임 경로 / 런처 경로

게임을 먼저 실행한 뒤 어시스턴트를 켜면 `게임 경로`는 대개 자동으로 잡힙니다. 수동으로 지정할 경우,

런처인 `miHoYo Launcher\launcher.exe`를 잘못 선택하지 않도록 주의하세요. 실제 게임 실행 파일은 `Star Rail Game\StarRail.exe`입니다.

HoYoverse 런처를 통해 게임 업데이트나 사전 다운로드를 하려면 `HoYoverse 런처 경로`도 추가로 설정해야 합니다.

#### 작업 완료 후

현재 버전은 `없음`, `종료`, `시스템 종료`, `절전`, `최대 절전`, `재시작`, `로그오프`, `디스플레이 끄기`, `스크립트 실행`, `반복` 등을 지원합니다.

`종료`는 게임 종료를 뜻합니다. `스크립트 실행`은 별도의 경로를 입력해야 하며, `시스템 종료`나 `절전` 같은 동작은 취소할 시간을 주기 위해 먼저 알림을 띄우고 잠시 대기합니다.

#### 예약 실행

`반복 모드`는 구버전 설정 호환을 위해 남아 있지만, 새 버전에서는 `로그` 페이지의 `예약 작업`을 쓰는 편이 더 좋습니다.

예약 작업은 다음을 지원합니다.

- `전체 실행`, `개척력 소모`, `화폐 전쟁` 같은 내장 작업 실행
- 외부 프로그램이나 스크립트 실행
- 개별 작업에 대한 타임아웃 및 강제 종료
- 완료 후 알림 전송
- 완료 후 동작 설정
- 완료 후 특정 프로세스 자동 종료

24시간 무인 운용을 한다면 구형 반복 모드보다 `로그 페이지 예약 작업`을 우선적으로 사용하는 것이 좋습니다.

### 개척력 및 일일 설정

#### 던전 종류 / 던전 이름

개척력 소모 시 어떤 던전을 돌지 결정하는 핵심 항목입니다. 현재 개척력을 기준으로 자동으로 횟수를 계산합니다.

`차원 분열` 같은 이벤트가 감지되면 여기서 지정한 던전을 우선 사용합니다.

#### 예비 개척력 사용 / 연료 사용

개척력 소모 시 예비 개척력 또는 연료를 추가로 사용할지 정하는 옵션입니다. 각 항목은 1회 사용 상한이 있습니다.

#### 지원 캐릭터

`지원 캐릭터 사용`을 켜면 필요할 때 지원 캐릭터를 시도하고, `지원 캐릭터 강제 사용`을 켜면 모든 던전에서 지원 캐릭터를 우선 사용합니다.

#### 육성 목표

`육성 목표`를 켜면 목표에 따라 행적 또는 유물 던전을 자동 매칭합니다. 목표를 인식하지 못하면 수동으로 설정한 기본 던전으로 돌아갑니다.

#### 전쟁의 여운

활성화하면 전체 실행이 매주 3회의 `전쟁의 여운`을 우선 처리하며, 어느 요일부터 시작할지도 설정할 수 있습니다.

### 화폐 전쟁 설정

매주 화폐 전쟁 점수 보상만 자동으로 챙기고 싶다면 `화폐 전쟁 점수 보상 활성화`만 켜면 됩니다.

자주 쓰는 설정 항목은 다음과 같습니다.

- `종류`: 표준 대결 / 오버클럭 대결
- `빠른 설정`
- `직급 난이도`
- `화폐 전쟁 전략`
- `특정 옵션 등장 시 재시작 허용`
- `속도전 모드 활성화`

### 차분 우주 / 시뮬레이션 우주 설정

차분 우주 주간 점수 보상만 자동으로 챙기고 싶다면 `차분 우주 점수 보상 활성화`를 켜면 됩니다.

`차분 우주` 페이지는 주간 점수 보상, 도감 해금, 레벨 상승을 담당하며,

일반 연산 / 주기 연산, 난이도, 저사양 호환 모드, 보상 후 장신구 추출, 스테이션 우선순위 등을 설정할 수 있습니다.

`시뮬레이션 우주 / 차분 우주` 페이지는 주당 100 엘리트 한도까지 유물 경험치와 영광의 눈물을 파밍하는 데 사용되며, 설정 항목은 다음과 같습니다.

- `실행 모드`: 통합 / 소스 코드 (시뮬레이션 우주만 해당)
- `종류`: 차분 우주 / 시뮬레이션 우주
- `실행 빈도`: 주간 / 일간
- `실행 횟수`
- `시뮬레이션 우주 몰입 보상 자동 수령`
- `운명의 길` (시뮬레이션 우주만 해당)
- `난이도` (시뮬레이션 우주만 해당)
- `시뮬레이션 우주 GPU 가속 사용` (시뮬레이션 우주만 해당)

`소스 코드` 모드를 사용할 경우 해당 PC에 사용 가능한 Python 환경이 있어야 합니다.

### 필드 파밍 설정

필드 파밍 페이지에서는 활성화 외에도 다음을 설정할 수 있습니다.

- `실행 모드`: 통합 / 소스 코드
- `자동 파티 전환`
- `맵 버전`
- `우선 행성`
- `비술 간식 구매 및 합성`
- `토큰 및 만료 우편 구매`

`소스 코드` 모드를 사용할 경우 해당 PC에 사용 가능한 Python 환경이 있어야 합니다.

### 추광줍기 설정

`혼돈의 기억`, `허구 이야기`, `종말의 환영`은 각각 다음을 별도로 설정할 수 있습니다.

- 활성화 여부
- 층 범위
- 두 파티 편성
- 마지막 실행 시간

먼저 팀 편성을 맞춰 놓고, 이후 전체 실행으로 주간 콘텐츠를 맡기는 것을 권장합니다.

### 클라우드 스타레일

`클라우드 스타레일 사용`을 켜면 어시스턴트가 브라우저로 클라우드 게임을 제어하여 개척력 소모 같은 작업을 실행합니다.

자주 쓰는 설정은 다음과 같습니다.

- `전체 화면 실행`
- `최대 대기 시간`
- `브라우저 종류`
- `브라우저 및 드라이버 국내 미러 사용`
- `브라우저 데이터 저장`
- `브라우저 화면 배율(DPI)`
- `무창 모드 활성화(백그라운드 실행)`
- `로그인되지 않았을 때 자동으로 창 모드 전환`

무창 모드는 시뮬레이션 우주(차분 우주 제외)와 필드 파밍을 지원하지 않습니다. 백그라운드에서 일일 숙제와 개척력 소모를 돌리고 싶다면 이 설정들이 매우 중요합니다.

### 기타 설정

`기타` 페이지에는 안정성과 자동화 경험을 높이기 위한 보조 스위치가 모여 있습니다. 대표적인 항목은 다음과 같습니다.

- `자동 전투 및 2배속 감지 활성화`: 게임 시작 전에 자동 전투와 2배속을 켜 보며, 개척력 소모나 화폐 전쟁 같은 내장 전투에서도 상태를 유지합니다.
- `자동 해상도 변경 및 Auto HDR 끄기`: 어시스턴트로 게임을 시작할 때 1920×1080으로 바꾸고 Auto HDR을 끕니다. 로컬 모드 사용자에게 적합합니다.
- `게임 경로 자동 구성 활성화`: 바로가기, 공식 런처, 실행 중인 게임 프로세스를 바탕으로 경로를 자동 감지합니다.
- `백그라운드 스크린샷 우선 사용`: 기본적으로 켜 두는 것을 권장합니다. 오버레이나 플로팅 창의 방해를 줄일 수 있습니다.
- `사용자 로그인 시 시작`: Windows 작업 스케줄러를 통해 로그인 시 자동 시작합니다. 완전 무인 운용을 하려면 시스템 자동 로그인도 별도로 설정해야 하는 경우가 많습니다.

`OCR 가속 모드`는 우선 `자동`으로 두는 것을 권장합니다. 각 모드는 다음과 같이 동작합니다.

- `자동`(기본값): Windows 10 18362 이상에서는 ONNXRuntime(DirectML) GPU 가속을 우선 사용합니다.
- 그렇지 않고 OpenVINO가 감지되면 OpenVINO CPU를 사용하며, 둘 다 아니면 ONNXRuntime CPU로 돌아갑니다.
- `GPU (ONNXRuntime DirectML)`: DirectML GPU 가속을 강제하며, 환경이 맞지 않으면 자동으로 되돌아갑니다.
- `CPU (OpenVINO)`: OpenVINO CPU 추론을 강제로 사용합니다. 순수 CPU 환경에서는 ONNXRuntime CPU보다 빠른 경우가 많지만,
- 현재 버전은 OpenVINO CPU runtime cache 문제에 대한 임시 우회책을 적용하고 있으며, 사용 가능한 물리 메모리가 1GB 미만으로 떨어지면 자동으로 ONNXRuntime CPU로 전환합니다.
- `CPU (ONNXRuntime)`: ONNXRuntime CPU 추론을 강제로 사용합니다. 호환성이 가장 좋고 메모리 동작도 가장 안정적입니다.
- 인식 이상, 크래시, 과도한 메모리 사용이 있다면 우선 `CPU (ONNXRuntime)`으로 바꿔서 확인하세요.

### 단축키 설정

단축키 설정은 `기타` 페이지에 있으며, 어시스턴트 내장 기능에만 적용됩니다. 비술, 맵, 워프, 작업 중지 같은 단축키를 설정할 수 있습니다.

이 중 `비술` 단축키는 개척력 소모와 추광줍기 같은 내장 작업에서 주로 사용됩니다.

시뮬레이션 우주나 필드 파밍의 단축키를 바꾸고 싶다면 해당 서드파티 프로젝트의 문서와 설정 파일을 참고하세요.

### 계정 관리 (Windows 전용)

`계정` 페이지는 Windows에서만 표시되며, 주로 로컬 게임 다중 계정 내보내기와 전환에 사용됩니다. `클라우드 스타레일`은 지원하지 않습니다.

권장 순서는 다음과 같습니다.

1. 게임에 완전히 들어가 캐릭터가 대세계에 서 있는 상태까지 진행합니다.
2. `현재 게임 계정 내보내기`를 눌러 현재 계정을 로컬 목록에 저장합니다.
3. 이후 전환이 필요할 때 목록에서 대상 계정을 선택하고 `선택한 계정 가져오기`를 누릅니다.

계정 페이지는 다음 작업도 지원합니다.

- `계정 이름 변경`: 저장한 계정에 알아보기 쉬운 이름 부여
- `계정 삭제`: 저장된 로컬 계정 데이터 삭제
- `새로 고침`: 로컬 계정 목록 다시 읽기
- `레지스트리 정리`: 현재 PC에 남아 있는 관련 계정 정보를 정리해 계정 전환 문제를 점검할 때 도움

`자동 로그인`은 해당 계정에 입력한 아이디와 비밀번호도 추가로 저장하고, 로그인 만료 시 자동으로 다시 채워 넣으려고 시도합니다.

이 기능은 어느 정도 위험이 있으므로, 정확히 무엇을 하는지 이해하고 있을 때만 사용하는 것을 권장합니다.

> 중요: 게임에 완전히 진입하기 전에 계정을 내보내면 이전 계정의 UID를 잘못 읽어서 저장 파일과 실제 계정이 맞지 않을 수 있습니다.
